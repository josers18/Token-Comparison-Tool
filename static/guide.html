User guide

A walkthrough of every screen in the tool, with screenshots of each surface and what to do on it.

1. Getting started

The first time you visit the app, you'll see a sign-in splash. The whole tool is gated on a Salesforce session — preflight verifies the dyno is healthy, but you also need to authenticate so the runner can call your org.

  1. Click Connect Salesforce → on the splash, or open the auth chip in the header (the pill that says "Sign in").
  2. You'll be redirected to your Salesforce org's login page. Approve the OAuth scopes (mcp_api, cdp_api, refresh_token).
  3. The popup closes and the header chip flips to ● Connected. The home chooser appears.

Tip: the auth chip is everywhere. Sign out from any page and the SPA snaps back to the splash; sign in and you land back on the home chooser.

2. The catalog

From the home chooser, click Run benchmark. The catalog shows six scenarios as a card grid — each card is a real Salesforce task ranging from a simple SOQL query (s01) to a multi-DMO Customer 360 join (s06).

  • Each card shows the scenario's category dot (core-crm = teal, data-360 = coral), id and title, difficulty pill, and a tiny sparkline of recent Native vs MCP cost trends.
  • Tri-state checkbox. Top-left "Select all" toggles every card; clicking individual cards' checkboxes adds/removes them from the run set.
  • Sticky controls panel at the bottom: pick the number of runs per path (1/3/5), one or more models (haiku / sonnet / opus), and the max-turns cap.
  • Click Run benchmark → when ready.

3. Running a benchmark

Once you click Run, the SPA opens a Server-Sent Events stream and the stepper appears (s01 / s02 / … / Summary). Each scenario runs both paths against every selected model, in random path order, and the UI updates live as runs complete.

  • Stepper chips turn solid as their scenario completes. Click any chip to jump into that scenario's detail mid-run.
  • Polling fallback. If SSE drops (corporate proxy, sleeping browser), the SPA falls back to polling /api/run/status with no visible jank.
  • Failures don't stop the run. An auth error, max-turns exhaustion, or rate-limit retry shows up in the scenario's outcome bar; the rest of the catalog continues.

4. Scenario detail

Click any scenario chip to drill in. This is where most of the analysis happens.

  • Verdict hero. Editorial Fraunces headline ("Native is 2.3× cheaper here.") with two callouts: dollars saved per 10,000 runs, and the input-token delta between the paths.
  • Sparkline KPI cards. Cost trend, cache hit, success rate, p95 wall-clock — each plotted as a recent Native (theme-native color) vs MCP (theme-mcp color) trend, when there are ≥2 reports to chart.
  • Native / MCP panels. Side-by-side glass cards: input-tokens hero number, turns, cost per run, cache hit, wall-clock, and the list of tools each path actually called.
  • Per-run breakdown. Every individual run that contributed to the medians, with succeeded/failed status. Click a row to expand the full input → tool calls → output replay.
  • Comparison chart. Custom horizontal bar chart with Δ percentages (Input tokens, Output tokens, Cost, Turns).
  • Why these numbers differ. Auto-generated editorial paragraph that explains the cost gap (e.g., "MCP loaded 16 tools at startup → 2,041 extra cache-create tokens in Native but two extra system tokens per turn for MCP").
  • Turn-by-turn trace. Every turn of the first successful run of each path, side-by-side. Cache writes on turn 1 are usually tool-definition loads.

5. Summary deck

The Summary chip at the end of the stepper aggregates every scenario into a one-page editorial recap.

  • Headline. Auto-generated executive sentence ("Across 6 scenarios, native integrations cost ~47% less per task than MCP — 1.9× cheaper on average").
  • Cost totals. Three glass cards (Total cost, Total input tokens, Success rate) with palette accent stripes; Native row in palette-native color, MCP row in palette-mcp color.
  • Cost at scale. Interactive log-scale slider (10 → 1,000,000 runs/month). See section 6.
  • Per-scenario bars. One row per scenario with side-by-side gradient bars and the cost ratio (Native vs MCP, "low conf." flag if only one run).
  • When Native wins / When MCP wins. Auto-classified framework grid showing which scenario shapes favor which path.
  • Run outcomes. Stacked bars showing succeeded vs max-turns vs auth-error vs no-tool-calls, per path.
  • Cache effectiveness. Two tiles with the cache-hit fraction per path. 0% means every input token was billed fresh.
  • Caveats. Auto-flagged warnings (single-run measurements, untested model, etc) with amber ! markers.

6. Cost-at-scale projection

Inside the summary, the projection panel lets you forecast monthly spend at any volume — useful for annual-budget conversations.

  • Volume slider, log-scale 10 → 1M monthly runs. Drag to any value; the API recomputes on rAF + 300ms debounce.
  • Period. Daily / Monthly / Annual segmented control. Defaults to Monthly.
  • Growth %. Optional month-over-month growth rate. Compounds across the 12-month projection curve.
  • Model. Picker if multiple models were run. Otherwise hidden.
  • Three KPIs: Native total, MCP total, Delta (with × cheaper).
  • Breakeven thresholds. Three editable dollar targets ($1k / $10k / $100k by default). The table shows the volume at which each path crosses each threshold per scenario.
  • 12-month curve. Native + MCP cumulative spend lines. The two lines show how the gap compounds over time.

7. Free-format scenario

Back on the home chooser, Free-format scenario opens a textarea so you can run a one-off prompt that isn't in the catalog yet.

  • Pick model, runs per path, and max turns independently.
  • Click Run free format →. The scenario gets its own indigo-dot tab in the stepper next to the catalog scenarios — it's distinguishable because catalog scenarios have a teal dot.
  • The same scenario detail UI works for free-format runs. Output isn't saved as a YAML, but the report row in Postgres carries the prompt verbatim so you can re-find it on the History page.

8. History & reports

The History link in the header opens a per-scenario, per-model trend walker — useful for "did our latest infra change help or hurt?". The Reports table on the home chooser's "View previous runs" card is the operational counterpart: every saved report sortable by every column.

  • Reports table. Sort by date / model / scenario count / cost / ratio / success / cache hit. Filter by kind (catalog / freeform / mixed) or by model. Search by report id, operator, or org.
  • Click a row to load the report into the same scenario / summary views as a fresh benchmark.
  • Actions menu on each row: Open in app, Download Markdown, Download PDF, Compare to another row.
  • History page (/history): 2×2 chart grid (cost / cache / success / p95) per scenario, per model, with change markers when a config tweak shifts the curve.

9. Comparing two reports

The Compare link (or the per-row Compare button on the reports table) opens a side-by-side diff page.

  1. Pick Report A and Report B from the dropdowns.
  2. Pick a model that both reports ran (the dropdown only shows shared models).
  3. Click Run comparison →.

The result groups scenarios into Regressions (cost up >10% or success down >5pp), Other scenarios (no significant change), and Scope changes (added or removed). Each scenario card shows the Native cost, MCP cost, success rate, cost multiplier, and p95 latency for both reports with the delta.

10. Sharing a report

From any scenario detail or summary view, click Share link.

  • The modal shows a copyable URL plus the expiry date (default 30 days).
  • Click Copy to put it on the clipboard. Regenerate mints a fresh token without revoking the old one (rotating SESSION_SECRET would invalidate every outstanding token in one go).
  • Recipients open the URL with no Salesforce login required. They get a read-only view of the same scenario / summary screens.
  • Slack / iMessage / Twitter unfurl the link as a server-rendered OG card showing the verdict and Native/MCP medians, themed to whichever palette the recipient is on (or the default if they've never visited).

11. Themes & preferences

The header carries two preference controls. Both persist across sessions and across pages.

  • Theme puck. Pill that shows the active theme (e.g., Teal · Light). Click to open a dropdown with a Light/Dark toggle, four palette tiles (Teal+Coral, Emerald+Violet, Cyan+Amber, Forest+Terracotta), and a "Match system color scheme" checkbox. Eight total looks; pick whichever you like best.
  • Auth chip. Pill showing your Salesforce status: Sign in when logged out, Connected with a breathing pulse when logged in. Click to sign out without leaving the page.
  • PDF export uses a print stylesheet that flattens animations and forces light mode regardless of your active theme — keeps reports legible on paper and in slide decks.
  • OG cards for share links pick up your theme via cookie. Recipients see a card themed to their own preference (or the default if they've never visited).
