Glossary

Metrics, terminology, and definitions used throughout the Token Comparison Tool.

Paths

Native (Path A)
Claude is given a hand-built set of REST tools (execute_soql, describe_object, list_sobjects, run_dc_query) and dispatches each tool_use block by calling Salesforce REST APIs directly from the runner. No MCP servers loaded.
MCP (Path B)
Claude is given the Salesforce-hosted MCP gateway via the Anthropic Messages API mcp_servers parameter. The gateway resolves tool calls server-side. Same prompt, same model, same max-turns cap as Native — only the tool provider differs.
Path order
Randomized per scenario to avoid systematic warm-cache bias toward whichever path runs second.

Token metrics

Input tokens
Total prompt tokens billed for a turn. Includes the system prompt, prior turns replayed back to the model, and any tool-result content.
Output tokens
Tokens generated by the model — the assistant's reply and any tool-use JSON it emitted.
Cache read input tokens
Of the input tokens, the subset that was served from Anthropic's prompt cache instead of being freshly billed. Higher = more turns hit cache → lower steady-state cost.
Cache creation input tokens
The first time a chunk of context (e.g., tool definitions) is sent to the model, Anthropic writes it to cache. Those tokens are billed at a slight premium but every subsequent turn that replays them counts as cache read and bills cheaper.
Cache hit rate
Fraction of input-side tokens served from cache vs freshly billed: cache_read / (cache_read + new_input). Reported per-path on the summary's "Cache effectiveness" tiles.
Token delta
The difference in input tokens between the MCP and Native paths for the same scenario. Negative means Native used fewer tokens.

Cost & performance

Cost / run
Median total USD cost across runs of one path. Computed from the SDK's usage object using a per-1M-token price table in pricing.py.
Cost ratio (× cheaper)
The ratio of MCP median cost to Native median cost. 2.3× means MCP costs 2.3 times what Native costs for the same scenario.
Wall-clock
End-to-end duration of a single run, in milliseconds, measured from the first messages.create to the final tool-result return.
p95 wall-clock
The 95th percentile of wall-clock duration across runs of one path. Smooths over outliers without hiding tail latency the way an average would.
Cost-at-scale projection
Monthly cost extrapolated to a target volume (10 → 1,000,000 runs/month) with optional growth rate. Drives the slider on the summary view; backed by /api/reports/{id}/projection.
Breakeven threshold
The volume at which one path's cumulative cost crosses a target dollar figure ($1k / $10k / $100k by default). Surfaces in the projection panel's threshold table.

Run outcomes

Succeeded
The model finished within the max-turns budget, the output passed the scenario's success_criteria (typically a substring match), and no API error occurred.
Max turns
The model used all turns in its budget without producing a final answer. Usually a sign the scenario is too complex for the path under test, or the path is doing too many tool round-trips.
Auth error
The Salesforce token was rejected (expired, missing scope, etc). The runner refreshes tokens before each run, so persistent auth errors point at OAuth scope misconfiguration.
No tool calls
The model declined to invoke any tool. Often the model "answered" purely from training data without checking the org. Counted as failed.
Inference error
The Anthropic API returned a non-2xx — rate limit, server error, malformed request. The runner retries once honoring Retry-After; persistent errors are recorded with the response body for replay.
MCP-unresolved tool use
The model emitted a tool_use block but the MCP gateway returned no matching tool. Usually an auth scope issue or a stale tool catalog.

Comparison terminology

Cube
The shape of a multi-model report. scenarios[].runs_by_model[model] holds Native + MCP runs per model. Legacy single-model reports are normalized to a one-model cube on read so every consumer gets the same shape.
Regression
On /compare: a scenario where report B's cost is >10% higher than A's, or success rate dropped >5pp. Surfaces first in the diff so reviewers see the pain immediately.
Scope change
A scenario added in B (didn't exist in A) or removed in B. Reported separately from regressions because the comparison is not apples-to-apples.
Champion
The reigning best-cost report on the History page. Pinned manually or auto-selected as the report with the lowest median Native cost across the full catalog.

Sharing & auth

Share token
An HMAC-SHA256 signed token (keyed on SESSION_SECRET) that grants read-only access to a single report for a configurable TTL (default 30 days). Verified on every /api/share/<token>/... call. Rotating the secret invalidates all outstanding share tokens.
Guest mode
How the SPA renders for share-link recipients. Sets window.__SHARE_GUEST__ = true and routes API calls through /api/share/<token>/.... Hides authenticated controls (Run benchmark, Admin) and any UI marked data-hide-in-guest.
OG card
Server-rendered 1200×630 PNG generated on demand from a share token. Slack / Twitter / iMessage unfurl shows the verdict + Native/MCP medians. Cached in-memory keyed by (token, theme, palette).
Preflight
The startup health check (/api/preflight) that verifies Heroku Inference, Postgres, and Salesforce OAuth are all reachable. Drives the READY indicator in the header.
Session
A server-side row in the Postgres sessions table holding the SF OAuth token. Keyed by an HTTP-only signed session cookie. Survives dyno restarts.
